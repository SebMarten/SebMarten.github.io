Data Exploration in SQL

For a small, hypothetical company, I populate, manipulate, and eventually clear a database involving sales representatives, products, and orders.

  • Check out the complete code in GitHub

In this SQL script, Definition Language (DDL) is employed to structure a relational database. Specifically, DDL commands create tables for sales representatives, products, and orders, each with specific fields and data types. The tables are designed with primary keys for unique identification and foreign keys to establish relationships between them. Additionally, the script utilizes DDL to alter the database structure by dropping tables at the end, ensuring a clean slate for future operations.

Example

-- Creating the orders table
CREATE TABLE orders (
	order_id SERIAL PRIMARY KEY,
	product_id INT REFERENCES products(product_id),
	rep_id INT REFERENCES sales_reps(rep_id),
	quantity INT,
	order_date DATE
);

In the provided SQL script, Data Manipulation Language (DML) is extensively used to interact with the database content. DML operations include inserting new records into tables for sales representatives, products, and orders, thus populating the database. It also encompasses updating specific records, such as adjusting product prices and sales representative details. Furthermore, DML is applied to retrieve information through SELECT queries, calculate aggregations, and delete records or entire sets of data from the tables, showcasing a comprehensive manipulation of the stored data.

In this SQL script, Data Query Language (DQL) is prominently utilized to query and retrieve data from the database, primarily through the SELECT statement. These queries range from simple selections, retrieving all records under certain conditions, to more complex ones involving joins, aggregations, and subqueries to analyze relationships and patterns within the data. By employing DQL, the script extracts meaningful insights such as product sales data, sales representative performance, and product pricing, showcasing the power of querying in data analysis and decision-making processes.

Example

-- Selects orders placed in March 2023
SELECT * FROM orders 
WHERE order_date 
BETWEEN '2023-03-01' 
AND '2023-03-31';
